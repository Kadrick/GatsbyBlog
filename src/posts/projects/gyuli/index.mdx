## 개요

### 기술 스택

- Java Spring Boot
- MySQL
- WeasyPrint (HTML → PDF 변환)

### 프로젝트 소개

| | |
| --- | --- |
| **개발 기간** | 2024.10 ~ 2024.12 |
| **역할** | 외주 진행 프로젝트 (Back-end 개발) |
| **목표** | 감귤 재배 농가의 영농 기록을 체계적으로 관리하고, 행정·지원 사업 증빙에 바로 쓸 수 있는 표준 양식 PDF 리포트를 생성하는 시스템 구축 |

감귤 재배 농가에서 매일 기록하는 영농일지를 모바일 앱과 통합 백엔드로 수집·관리하는 시스템을 개발했다. 기후 데이터와 품종별 재배 환경을 데이터베이스화해 농가별·기간별 조회가 가능하도록 했고, 작성된 일지를 **WeasyPrint** 오픈소스를 활용해 표준 양식의 PDF 리포트로 변환하는 모듈을 구현했다.

## 작업 내용

### 통합 백엔드 · API

- **REST API 설계** — 모바일 앱·관리자 대시보드가 공유하는 단일 API 계층 구성
- **DB 스키마 설계** — 농가, 품종, 기후·재배 환경, 영농일지, PDF 생성 이력 테이블 정의

### 영농일지 기록 · 조회

- **일지 작성 API** — 농가별 작업 내용, 투입 자재, 기후·생육 상태 기록
- **품종별 재배 환경 관리** — 품종에 따른 재배 조건·기준 데이터를 사전 등록해 일지 작성 시 참조
- **기간별 · 품종별 조회** — 농가·기간·품종 조건으로 일지 목록 및 상세 데이터 조회

### 관리자 대시보드

- **농가 등록 · 현황 조회** — 등록된 농가 목록과 일지 작성 현황 모니터링
- **기간별 · 품종별 통계** — 관리자가 농가별 데이터를 집계·분석할 수 있는 조회 화면 제공

### PDF 리포트 변환 (WeasyPrint)

- **HTML 템플릿 작성** — 행정·지원 사업 증빙 양식에 맞는 표준 HTML·CSS 템플릿을 직접 작성
- **데이터 삽입** — 조회한 일지 데이터를 HTML 템플릿에 삽입해 완성된 HTML 문서 생성
- **WeasyPrint 연동** — 생성된 HTML을 WeasyPrint로 PDF 변환
- **한글 폰트 임베딩** — CSS `@font-face`로 한글 폰트를 지정해 PDF 출력 시 글리프 깨짐 방지
- **PDF 생성 API** — 일지 ID 기준으로 PDF를 생성·다운로드하는 엔드포인트 제공

## 설계 포인트

### WeasyPrint를 선택한 이유

영농일지 PDF는 농가마다 내용은 다르지만 **레이아웃과 양식은 동일**해야 한다. iText나 Apache PDFBox처럼 좌표 단위로 PDF를 직접 그리는 방식은 양식 변경 시 코드 수정 비용이 크다. WeasyPrint는 HTML/CSS로 레이아웃을 정의하고 이를 PDF로 변환하므로, 웹에서 쓰는 템플릿 기법을 그대로 활용할 수 있다.

### HTML 템플릿 기반 리포트 생성

표준 양식에 맞는 HTML·CSS 템플릿을 직접 작성하고, PDF 생성 시 일지 데이터를 템플릿에 삽입해 완성된 HTML을 만든 뒤 WeasyPrint로 변환하는 2단계 파이프라인으로 구성했다. 표·헤더·푸터 레이아웃은 CSS로 관리하므로, 행정 양식이 바뀌면 HTML 템플릿 파일만 수정하면 된다.

### 한글 폰트 처리

WeasyPrint는 시스템 폰트에 의존하지 않고 CSS에서 명시한 폰트를 PDF에 임베딩한다. 한글 문서에서 흔한 글리프 누락·깨짐을 방지하기 위해 `@font-face`로 TTF 폰트를 지정하고, 템플릿 전역에 `font-family`를 적용했다. ([PDF 파일 구조와 텍스트 추출](/post/computer/common/pdf-format)에서 다룬 CJK 폰트 이슈와 달리, 생성 단계에서는 폰트 임베딩으로 해결 가능하다.)
