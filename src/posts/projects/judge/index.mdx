## 개요

### 기술 스택

- TypeScript, React
- Go
- PostgreSQL
- RabbitMQ
- GitHub Actions
- Docker

### 프로젝트 소개

- **개발 기간:** 2024.08 ~ 2026.08 (운영 중)
- **역할:** 1인 개발 (학부 연구생 참여 후 시작)
- **목표:** 교내 강의에서 사용하는 온라인 채점 시스템 — 코드 채점, 문제 제출, 성적 통계 제공

<CaseStudy
  id="case-1-rabbitmq-eda"
  title="Case 1. RabbitMQ 기반 이벤트 기반 아키텍처(EDA) 전환"
  problem={
    <>
      실습 마감 직전 집중되는 대규모 트래픽으로 모놀리식 구조의 API 서버가 다운되었다.
      채점 요청과 실행 로직이 동기적으로 묶여 있어 피크 타임에 병목이 발생했다.
    </>
  }
  action={
    <>
      RabbitMQ 메시지 큐를 도입하여 채점 요청 수신과 실제 실행 로직을 분리했다.
      API는 요청을 큐에 publish하고, Judge Worker가 비동기로 소비하는 구조로 개편했다.
    </>
  }
  application={
    <>
      채점 요청 → 큐 적재 → Worker pool 분산 처리 파이프라인을 구성했다.
    </>
  }
  result={
    <>
      피크 타임 병목 현상을 제거했고, 120명 이상 동시 접속 상황에서도 무중단 운영을 달성했다.
    </>
  }
/>

<Mermaid chart={`sequenceDiagram

<CaseStudy
  id="case-2-cursor-partitioning"
  title="Case 2. Cursor 페이지네이션 및 수평 파티셔닝을 통한 성능 최적화"
  problem={
    <>
      2년간 누적된 제출·채점 데이터로 인해 조회 API 응답이 3~4초까지 지연되었다.
      OFFSET 기반 페이지네이션은 데이터가 커질수록 스캔 비용이 급증했다.
    </>
  }
  action={
    <>
      OFFSET 대신 Cursor(keyset) 기반 페이지네이션으로 전환했다.
      기간별 수평 파티셔닝을 적용하여 조회 범위를 물리적으로 분리했다.
    </>
  }
  result={
    <>
      1,000회 테스트 기준 평균 응답 시간을 4초에서 0.5초로 단축했다 (약 87% 개선).
    </>
  }
/>
