공장 현장에서 근무자의 안전을 지원하기 위한 Android 앱과 이를 뒷받침하는 백엔드·관리자 대시보드를 개발했다. 위험구역 GPS 감지, QR 기반 화학물질 조회, 안전보건 자료 제공, 제보·공지 기능을 하나의 통합 백엔드로 묶어 운영한다.

---

## 작업 내용

### 통합 백엔드 · API

- **REST API 설계** — 모바일 앱·관리자 대시보드가 공유하는 단일 API 계층 구성
- **DB 스키마 설계** — 위험물, 위험구역, 화학물질, 안전 자료, 제보·공지, 진입 이력 테이블 정의

### GPS 기반 위험구역 감지

- **위험구역 좌표 관리** — 관리자가 위험물별 위험구역 좌표를 사전 등록
- **단말기 연동** — 앱이 서버로부터 위험구역 정보를 수신해 GPS 현재 좌표와 대조
- **경보 · 알림** — 위험구역 진입 시 경보음 발생 및 위험 신호 푸시 알림 전송
- **진입 이력 수집** — 알림 발생과 동시에 서버로 진입 이력 전송

### QR 화학물질 조회

- **QR 코드 생성** — 화학물질별 상세 조회용 QR 코드 생성 모듈
- **스캔 조회 API** — 앱 QR 스캔 결과를 화학물질 상세 정보로 연결

### 현장 안전 기능

- **안전보건 자료** — MSDS 전문·요약본, 안전보호구 착용 기준 제공
- **안전 제보** — 현장 안전 이슈 제보 접수
- **공지사항** — 현장별 공지 게시·조회

### 관리자 대시보드

- **운영 현황 조회** — 현장별 위험구역 진입 로그, 알림 이력 확인
- **알림 정책 설정** — 현장별·구역별 Push/Sound 수신 정책을 동적으로 제어

## 설계 포인트

### 통합 백엔드로 기능 일원화

위험구역 감지, QR 조회, 안전 자료·제보·공지가 각각 분리된 API로 흩어지지 않도록 단일 Spring Boot 백엔드에서 처리했다. 모바일 앱과 관리자 대시보드가 동일한 데이터 모델을 공유하므로, 위험물 정보를 등록하면 앱의 GPS 감지·QR 조회·안전 자료 화면에 일관되게 반영된다.

### 현장별 알림 정책 분리

공장 현장마다 소음 수준과 작업 환경이 달라, 경보음·푸시 알림을 일괄 적용하기 어렵다. 현장별·구역별로 Push/Sound 수신 정책을 관리자가 동적으로 설정할 수 있도록 했고, 정책 변경이 앱 단말기에 반영되는 구조로 설계했다.

### 진입 감지와 이력 전송의 동시 처리

위험구역 진입은 현장에서 즉각 대응이 필요한 이벤트다. 단말기에서 경보음·알림을 발생시키는 것과 서버로 진입 이력을 전송하는 것을 동시에 수행하도록 흐름을 구성해, 근무자에게는 즉시 경고를 주고 관리자 대시보드에는 추적 가능한 기록을 남긴다.
