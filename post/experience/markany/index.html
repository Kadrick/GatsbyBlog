업무

워터마크 CMS 제품

배경

미디어 워터마크 삽입·관리는 회사가 고객에게 제공하는 상용 제품이었다. 고객 요구에 맞춰 기능을 확장하고 운영 안정성을 유지해야 했고, 기존 Vue 기반 프론트엔드와 함께 AWS 위 신규 아키텍처로의 전환·구축이 필요했다.

담당

  • 기존 Vue 프론트엔드를 React로 전환하는 작업
  • Go 백엔드와 연동하는 CMS 제품 유지보수 및 신규 구축
  • 워터마크 삽입 워크플로를 AWS 서비스와 연동하는 인프라 구성

핵심 구현

  • 고객이 콘텐츠를 업로드하면 워터마크 처리 후 결과를 S3에 저장하는 비동기 파이프라인 구축
  • Lambda·Step Functions로 처리 단계를 오케스트레이션하고, API Gateway·RDS로 API·메타데이터 관리
  • React UI에서 업로드 요청·처리 상태 모니터링·결과 확인이 가능하도록 제품 화면 개선

결과

상용 워터마크 CMS 제품의 유지보수와 신규 구축에 기여했고, 프론트엔드 전환과 풀스택 웹 서비스·AWS 운영 경험을 쌓았다.

Widevine DRM 스트리밍 PoC

배경

프리미엄 미디어를 HLS/DASH로 안전하게 송출하려면, 브라우저 재생(EME)부터 인코딩·패키징·DRM·라이선스 발급·CDN 배포까지 전 구간이 맞물려 동작하는지 검증해야 했다. 개별 기술을 이해하는 것과 실제 파이프라인으로 묶는 것은 별개의 문제였다.

담당

  • W3C MSE·EME 스펙 조사 및 데모 플레이어·라이선스 서버 직접 작성
  • MediaConvert · MediaPackage · CloudFront · Lambda@Edge 설정 및 연동
  • MediaPackage가 호출할 커스텀 라이선스 서버 개발
  • CloudFormation·Step Functions로 인프라·배포 단계 자동화

핵심 구현

  • 원본 영상 인코딩 → Widevine DRM 패키징 → CDN 배포 → 브라우저 EME 재생 → 라이선스 키 발급까지 PoC 파이프라인 구성
  • 브라우저 측에서는 MSE로 세그먼트를 조립하고 EME로 복호화 키를 요청하는 흐름을 데모 플레이어로 검증
  • AWS 측에서는 MediaPackage가 생성한 매니페스트·세그먼트를 CloudFront로 배포하고, 재생 시 라이선스 서버와 키 교환

관련 글

  • 영상은 어떻게 저장되고, HLS·DASH는 무엇인가
  • HLS와 DASH — 적응형 스트리밍 포맷 이해하기
  • MSE와 EME — 브라우저에서 암호화 미디어 재생하기
  • AWS로 Widevine DRM 스트리밍 파이프라인 구축하기
  • AWS Lambda에 VPC 설정하기

결과

암호화된 HLS/DASH 스트리밍 재생을 end-to-end로 검증했고, 미디어 포맷·브라우저 DRM API·AWS 미디어 서비스에 대한 이해를 확보했다.

PDF 워터마크 — 텍스트 추출

배경

PDF 문서에 워터마크를 넣으려면 본문 텍스트의 위치와 내용을 알아야 한다. 기존 텍스트와 겹치지 않게 배치하려면 좌표 정보까지 필요했지만, 상용 라이브러리만으로는 제품이 요구하는 수준의 추출·인코딩 처리가 어려웠다.

담당

  • PDF 파일 구조를 직접 파싱하는 구조 파서 개발
  • 텍스트 스트림에서 글리프 데이터를 추출하는 텍스트 추출기 개발
  • CMap 기반 CID 매핑·한글 인코딩 등 예외 구간 검증

핵심 구현

  • PDF 객체 트리·크로스 레퍼런스 테이블을 따라가며 페이지 콘텐츠 스트림에 접근
  • Tj·TJ 등 텍스트 연산자를 파싱해 문자열·좌표·폰트 정보 수집
  • 추출 결과(텍스트 + 좌표)를 PDF 워터마크 제품 PoC의 입력 데이터로 전달

관련 글

  • PDF 파일 구조와 텍스트 추출
  • Codepage, 그리고 한글 인코딩

결과

PDF 내부 구조를 이해하고 텍스트 추출 파이프라인을 확보했으며, 이후 워터마크 삽입 기능 개발의 기반이 되었다.
